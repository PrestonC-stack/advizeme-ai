Project Overview

Internal shop execution system for Callahan Auto & Diesel focused on improving inspection quality (DVI), estimate accuracy, technician documentation, and advisor decision-making. The system is intended to standardize how vehicle findings are captured, translated into estimates, and prioritized for repair, while reducing missed items, poor communication, and profit leakage.

Primary users:

Technicians (data input, inspections, diagnostics)
Advisors (estimate building, sales, prioritization)
Shop owner (oversight, audit, profitability control)

This project acts as the operational intelligence layer between DVI → Estimate → Customer approval → Repair execution.

Core Objective

Create a system that:

Ensures every inspection result is properly documented, justified, and converted into a complete, sellable estimate
Provides real-time coaching and validation for technicians and advisors
Eliminates missed work, poor documentation, and inconsistent processes
Improves profitability, clarity, and workflow efficiency across the shop

DVI / Inspection Standards

Good DVI includes:

Clear condition statement
NOT: “bad”
YES: “Front brake pads at 2mm, below spec”
Photos:
Close-up + context shot
Measurement values where applicable
Cause + effect explained
Recommendation tied to condition

Structure:

What is wrong
Why it matters
What should be done
Estimate / Audit Standards

Good estimate:

Every finding accounted for:
Sold
Deferred
Not applicable (with reason)
Labor matches scope
Parts complete (no missing components)
No vague line items
Proper tax handling (8.3% on parts/shop supplies)

Audit flags:

Missing labor for known tasks
Parts mentioned but not billed
DVI items not quoted
Underpriced labor
